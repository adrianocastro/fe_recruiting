Please take a look at the attached GIF image. The highlighted area is the Search Module. The intended functionality is:

The user must fill out all fields and hit "Find My Match" to submit a search.

If the Age or City/Zip fields are empty, return an error. If the first Age field is greater than the latter Age field, return an error. The City/Zip field can only contain valid characters (alphanumeric and special characters like periods [.], apostrophes ['], hyphens [-], and commas [,]).

All form validation should be client-side. The error message should indicate all of the current errors and place the user's focus back onto the first incorrect form field. You can name the form and fields whatever you'd like.

Field values:
+ I'm a: [Man] [Woman]
+ Seeking a: [Woman] [Man]
+ Located within: [50 miles] [only in the city] [5 miles] [10 miles] [15 miles] [25 miles] [100 miles] [250 miles]

Field defaults:
+ I'm a: [Man]
+ Seeking a: [Woman]
+ Age: [20] to [40]
+ Located within: [50 miles]
+ Show photo profiles only: [checked]

Build this functionality using cross-browser client-side code (for IE5+/NS7.1+). Optimize for download & render speed.
