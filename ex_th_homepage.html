Take-Home Exercise

Using the attached screenshots please build this as a single page.

The code should be semantically correct and display with visual fidelity in IE6+, FF3.5+, Chrome and Safari 5. It should degrade to a non-stylized layout that still retains some form of information hierarchy and usability for older browsers. It should also be accessible and retain its aspect ratio if the user changes the browser or platform’s text size settings. The page should be optimised for download and render speed. For the behaviour layer you’re welcome to use a JavaScript framework but, where possible, please use pure JavaScript.

Please note:

  • The center and right columns should be at least 800 pixels high
  • Most of the backgrounds are gradients
  • Some elements have drop-shadows

When you’re finished, please send us the code and all associated files. You may slice images from these mockups.

Base page

Mail hover

The Mail hover should open when the user mouses over the Mail row and remain open until the user clicks on the closing X icon or mouses over the Finance row. Clicking on the mail row should redirect the user to http://mail.yahoo.com/.

Hitting the "Sign up Now" submit button should validate the form as per the following rules:

  • all fields must be filled
  • the email address must be valid
Upon successfully validating the form should be replaced with the following text: 
        Hi <first_name>,
        
        Thank you for signing up!
Where <first_name> should be the user’s provided first name. Please replace only the form, the rest of the content should stay in place.

Finance hover

The Finance hover open/close interaction should be the same as the Mail hover. Clicking on the Finance row should redirect the user to http://finance.yahoo.com/. For extra points the content inside should be loaded via AJAX.
