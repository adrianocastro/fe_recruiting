Please take a look at the attached GIF image. The intended functionality is:

The user must fill out all fields and hit "Submit T" to submit a search.

For the Age fields, return an appropriate error message if:
+ Either field is empty
+ The values are not numbers
+ The value of the first field is greater than the latter
+ Either value exceeds 100

For the City/Zip field, return an appropriate error message if:
+ The field is empty
+ If a zip code is entered, if the zip is invalid (not 5 characters or 10 with a dash)
+ If a city is entered, if the value is an invalid city name (not alphabetic with special characters: dash [-] period [.] apostrophe ['] and space [ ])

Use both client-side and server-side validation. For client-side errors, use an Alert dialogue to display all of the relevant errors and have the focus brought back to the first field with an error. For server-side errors, reload the page and display the relevant errors above the module.

If no errors are present, the subsequent page should list out the user's selections. If the user returns to the Search Module (even through a new browser session), the previous data should be retained.

All of the field values are:
+ I'm a: [Man] [Woman]
+ Seeking a: [Woman] [Man]
+ Located within: [50 miles] [only in the city] [5 miles] [10 miles] [15 miles] [25 miles] [100 miles] [250 miles]

The default field values are:
+ I'm a: [Man]
+ Seeking a: [Woman]
+ Age: [20] to [40]
+ Located within: [50 miles]
+ Show photo profiles only: [checked]

Build this functionality using cross-browser client-side code (for IE5+/NS7.1+), PHP, and a MySQL database. Optimize for download & render speed and server performance.
