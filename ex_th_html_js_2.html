Please take a look at the attached GIF image. This is a truncated version of the current New User registration form. Please build the following validation functionality:

The user must fill out all fields and hit "Submit This Form" to submit the data.

For the First and Last Name fields, return an error if:
+ Either field is empty
+ The values are invalid (only allow alphabetic and the following special characters: dash [-] period [.] apostrophe ['] and space [ ])

For the Yahoo1 ID field, return an error if:
+ The field is empty
+ The value is invalid (only allow alphanumeric and the following special characters: underscore (_) dash [-] and period [.])

For the Password fields, return an error if:
+ Either field is empty
+ The value is invalid (only allow alphanumeric and the following special characters: underscore (_) dash [-] and period [.])
+ Both fields do not match exactly (the values are case-sensitive)

For errors, use an Alert dialogue to display all of the relevant error messages and have the focus brought back to the first field with an error. Then change the background of all invalid form fields to yellow. Do this without refreshing the page.

Build this functionality using cross-browser client-side code (for IE5+/NS7.1+). Optimize for download & render speed.
