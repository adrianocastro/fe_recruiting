Q: What is the difference between include & include_once? include & require?
A: If a file called with include_once has already been included, it will not be included again. include produces a warning when it fails, require produces a fatal error (halting execution).

Q: What is the difference between a reference and a regular variable? How do you pass by reference & why would you want to?
A: Make sure the candidate understand basic references and their usage. You only need to pass by reference if you need to modify the passed-in value. Note that it does not save memory or improve performance to pass by reference.

Q: What is the difference between foo() & @foo()?

Q: What is the value of $my_var after the function call? 

	$my_var = 10;
	function bar($zoo) {
	  global  $my_var;
	  if (count($zoo)) {
	    unset($my_var);
	  }
	}
	bar(array(1,2));
A: $my_var will equal 10. unset, inside a function, will only destroy the local variable. To destroy the global version, you would have to call unset($GLOBALS['my_var']).

Q: Given a line of text $string, how would you write a regular expression to strip all the HTML tags from it?
A: PHP supports POSIX Extended REs and Perl-compatible ones. Either is fine. 

	In POSIX:
	$string = ereg_replace("<[^>]*>", "", $string); 
	In Perl-compatible:
	$string = preg_replace("/<[^>]*>/", "", $string);
Extra credit if they mention you should use strip_tags.

A: Using serialize and unserialize is often all that is needed. Other ways include WDDX & var_dump.

Q: Write a class that represents a stack. The stack must have the following: a push method, a pop method, and a clear method. For the stack of this example, you'll most likely need an array, but don't assume array operators (array_push, array_pop, sizeof, count, etc. exist)
A: 

	class Stack {
	  var $_stack;
	  var $_count;
	  
	  function Stack() {
	    $this->clear();
	  }
	
	  function clear() {
	    $this->_stack = array();
	    $this->_count = 0;
	  }
	
	  function push($item) {
	    $this->_stack[ $this->_count ] = $item;
	    $this->_count++;
	  }
	
	  function pop() {
	    if ($this->_count > 0) {
	      $this->_count--;
	      return $this->_stack[ $this->_count ];
	    } else {
	      return null;
	    }
	  }
	}

Q: What would some of the challenges be in extending the class to keep track of the number of stacks created?
A: Two obstacles would be PHP's class/static data to track the total number of stacks (although the should be able to find a work around), and PHP's lack of destructors.
