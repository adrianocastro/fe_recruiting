Frontend Engineer Recruiting Process

This is a documentation of a recommended frontend engineer recruiting process.

Talent Identification

  1. Determine Skill Levels

    1. The skills of a frontend engineer can be placed into three main categories:

      Style & Layout (CSS)

      Since CSS is such a new technology, writing and debugging it is a creative, investigative task. The conceptual model of CSS is very different from markup or scripting languages. Strong CSS developers will also understand the box model and cross-browser issues.

      Skills include are: HTML, XHTML, CSS1, CSS2, specificity, inheritance, selectors, pseudo-classes, pseudo-elements, box model issues, display types, float rules, positioning rules, units, doctypes, quirks mode, standards mode, web standards, semantic markup, cross-browser hacks & filters, accessibility, internationalization, optimization.

      DOM Scripting (JS)

      JavaScript emphasizes event-driven programming, where the control flow of its scripts are largely driven by browser events, and DOM manipulation, where nodes can be created, modified, and removed in memory and without a page refresh. Strong JS developers will also understand cross-browser issues.

      Skills include are: JavaScript 1.2, JavaScript 1.5, ECMAScript, DOM0, DOM1, DOM2, programming fundamentals, data types, data manipulation, regular expressions, node manipulation, dynamic content retrieval, object-oriented programming, event-driven programming, event models, cross-browser issues, optimization.

      Server-side Integration (PHP)

      PHP is the primary server-side integration language used here. Object, array, and string manipulation are the main tasks used in the presentation layer of PHP. Strong PHP developers will also understand HTTP, get/post, and file caching.

      Skills include are: PHP4, PHP5, MySQL, Apache, Unix, command-line programming, HTTP headers, get/post requests, file caching, programming fundamentals, data types, data manipulation, regular expressions, object-oriented programming, database connectivity, state management, version control, internationalization, optimization.

    2. Determine the skill level required for each of the three skill categories:

      • Level 0: none - no knowledge or experience with this skill
      • Level 1: beginner/junior - some working knowledge, can modify existing code, needs guidance and frequent code reviews
      • Level 2: intermediate/mid - several years of experience with this skill, can write own code, can be an independent worker
      • Level 3: advanced/senior - strong experience with advanced concepts, go-to person and mentor, can lead code reviews

      Examples:

      Position One     Position Two
CSS: 1           CSS: 3
JS:  2           JS:  1
PHP: 2           PHP: 1

  2. Write Job Description

    Write a property-specific job description. Include property-specific qualifications and any distinguishing qualities or exciting opportunities that can help attract candidates to your position.

    Examples:

    • Frontend Engineer - updated generic description
    • Frontend Engineer for Yahoo.com
    • Frontend Engineer for Registration & Core Services
    • Frontend Engineer for Communities
    • Frontend Engineer for Yahoo! Photos

  3. Publish Job Description

    Work with HR to post job description on careers.yahoo.com and other job listing sites. Track your candidates on Resumix. Although headhunter/talent agencies may be used to increase the candidate flow, most of them charge exorbitant finders fees and Yahoo may not be able afford them.

Candidate Identification

  1. Review Resumes

    Assess each resume and rate the candidate based on the skill levels above. This is just a preliminary estimation; each candidate's ratings will change and be updated as he/she progresses through the recruiting process. These ratings should be used as a qualifier for the next step.

    Examples:

    • Resume assessment guidelines
    Candidate One     Candidate Two     Candidate Three     Candidate Four
CSS: 1            CSS: 3            CSS: 2              CSS: 1
JS:  2            JS:  1            JS:  2              JS:  0
PHP: 2            PHP: 0            PHP: 1              PHP: 0

  2. Phone Screen

    Contact candidates by phone (or email if a phone number isn't available). Phone screens should be short, between 15-30 minutes only. The goal is to get a more accurate rating of each skill and confirm the candidate's interest in the position.

    1. Briefly introduce yourself, the company, the position requirements and responsibilities
    2. Confirm the interest for the role
    3. Ask high-level technical questions to gauge the candidate's skill level
    4. Ask for an email of code samples and URLs of previous work
    5. (Optional) Tell the candidate about the take-home exercise if you want him/her to do it
    6. Tell the candidate about the next steps

    Revise the candidate's skill ratings (usually, the ratings will drop after your phone screen discovers that they don't know as much as you thought they did, although occasionally you'll find a star candidate).

    Examples:

    • Sample phone screen questions
    Candidate One     Candidate Two     Candidate Three     Candidate Four
CSS: 1            CSS: 2            CSS: 3              CSS: 1
JS:  1            JS:  1            JS:  3              JS:  0
PHP: 1            PHP: 1            PHP: 2              PHP: 0

  3. Send Take-Home Exercise

    This step is optional, though some hiring managers may wish to make it required in order to normalize each candidate's technical abilities. Choose a relevant take-home exercise or create one yourself that best matches the skill needs. Ideally, the exercise should match the property for which this candidate is interviewing. These exercises also should be updated periodically so candidates don't publish them to the public. Email it to the candidate and give him/her a timeline of one week.

    Examples:

    • HTML/CSS v1
    • HTML/CSS v2
    • HTML/CSS v3
    • HTML/CSS/DHTML
    • HTML/DHTML
    • HTML/JS v1
    • HTML/JS v2
    • PHP/DHTML
    • PHP/JS v1
    • PHP/JS v2

  4. Review Code

    Assess the candidate's code samples, URLs, and take-home exercise (if assigned). Look for practical applications the skills in which you need. Sometimes, a candidate will be able to surprise you by quickly learning a previously unknown skill. As you review his/her code, consider these factors:

    • Compatibility on different browsers
    • Appearance of JavaScript/PHP errors
    • Understanding of web standards
    • Understanding of accessibility
    • Understanding of internationalization
    • Clean code grammar (understanding of data types, use of control structures, etc)
    • Knowledge of advanced concepts (object-oriented programming, data abstraction, separation of code layers, etc)
    • Use of borrowed code from elsewhere (Dreamweaver, public domain scripts, etc)
    • Download and rendering performance optimization

    If the candidate completed the take-home exercise, also consider these factors:

    • Adherence to instructions
    • Visual fidelity to designs
    • Above-and-beyond embellishments

    Revise the candidate's skill ratings again. You should be getting a more and more accurate assessment of each candidate's skills by this time. Remove those that don't pass and move the qualified ones to the interview stage.

    Candidate One     Candidate Two     Candidate Three     Candidate Four
CSS: 1            CSS: 3            CSS: 3              CSS: 1
JS:  1            JS:  1            JS:  3              JS:  0
PHP: 1            PHP: 1            PHP: 2              PHP: 0

Interview

  1. Invite Candidate for 1st Round

    Call the candidate and invite him/her in for a 1st round interview. This interview should only be for an hour or so. The goal is to be able to make a preliminary HR grade/level assessment (e.g. L7 low, L9 high), delve deeper into technical competency, and assess potential team fit.

    For a second opinion, you can opt to have a lead or senior member of your team pair up with you, or have him/her interview the candidate right after you're done.

    Example:

    • 1st Round Interview Questions

    Make a quick assessment of the candidate.

    • Potential grade/level?
    • Potential technical competency?
    • Potential team fit?

    Revise the candidate's skill ratings one more time. Remove those that don't pass and move the qualified ones to the interview stage.

    Candidate One     Candidate Two     Candidate Three     Candidate Four
CSS: 1            CSS: 1            CSS: 3              CSS: 1
JS:  1            JS:  1            JS:  3              JS:  0
PHP: 1            PHP: 1            PHP: 2              PHP: 0

  2. Invite Candidate for 2nd Round

    Call the candidate and invite him/her in for an interview. You can have HR schedule the interview for you or do it yourself. To have HR schedule an interview, submit an Interview Request to your interview scheduler. If you invite the candidate yourself, send him/her a description of the interview so the candidate can adequately prepare for it.

    Example:

    • Interview Request to HR
    • Interview Description to candidate (with take-home exercise)
    • Interview Description to candidate (without take-home exercise)

  3. Schedule Interview

    If you are handling the interview schedule yourself, you will have to check for each interviewer's availability and reserve a conference room that can hold up to 4 people. An HR recruiter can easily do this for you though.

    Example:

    1:00 to 1:30 - Take-home Exercise Code Review : frontend eng group
1:30 to 2:30 - In-house Exercise
2:30 to 3:00 - In-house Exercise Code Review  : frontend eng group
3:00 to 3:30 - Interview Slot                 : product representative
3:30 to 4:00 - Interview Slot                 : backend eng representative
4:00 to 4:30 - Interview Slot                 : UED representative

  4. Interview Candidate

    The Code Reviews will provide practical examples of current knowledge. If the reviews end early or don't provide enough information, additional technical questions can be asked.

    Examples:

    • Interview Questions for CSS
    • Interview Questions for JS/DHTML/DOM
    • Interview Questions for PHP

  5. Assess Candidate

    Asses candidate at Candidate Review session to make a recruiting decision:

    • No - Low interest, below average candidate
    • Wait - Average candidate, backburner and continue to interview other candidates
    • Proceed - Good candidate, additional review/interviews if necessary
    • Hire - Excellent candidate, high interest, fast-track candidate (a rare rating to receive)
