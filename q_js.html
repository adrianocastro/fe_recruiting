Q: Is JavaScript intrepreted or compiled in/by the browser?
A: Intrepreted

Q: Is JavaScript strongly or weakly typed? (bonus: why?)
A: Weakly typed (because variables cannot be cast as a particular data type)

Q: Which browser supports attachEvent and/or addEventListener?
A: attachEvent: IE, addEventListener: Mozilla/Netscape

Q: What are all of the ways that an event can be binded to an HTML element? What are the advantages and disadvantages to each method of assignment?
A: attachEvent, addEventListener, document.object=event, <html onevent="">. First two allow multiple event bindings, but are browser-specific. Second allows a specification of bubble or trickle event movement. Third and fourth are x-browser but only allow one binding. Fourth also mixes logic with markup.

Q: Can an HTML Object be extended using the prototype keyword?
A: No, The HTMLElement Object is not native; the element can be extended using Object.someName = function(){}

Q: How is garbage collection handled in JS?
A: Garbage collection is handled using a mark-and-sweep collection method.

Q: How can one object inherit the properties and methods of another?
A: This is done using the prototype keyword in the following manner. function x(){ return; } x.prototype = new y(); function y(){ this.val = 0;return; } y.prototype.getVal = function(){ return val;} y.prototype.setVal = function(input){ val = input;} var myObject = new x(); myObject.getVal() // returns 0;

	<ul>
		<li>one</li>
		<li>two</li>

		<li>four</li>
	</ul>

Q: Consider the following HTML: 

	<form>
		<div>
			<input type="button" name="changediv" value="Change Div" onclick="ChangeDiv(this);">
		</div>	
	</form>
Without changing the above HTML, write/implement the function "ChangeDiv()" that will change the background color of the containing DIV to "#cccccc".

Q: How would you design an OO implementation that allows developer to instantiate a Panel object that can be dragged and dropped on the screen? Show the best implementation using JS, DOM, OO design, and no markup. Focus on the constructor and how would you tie the DOM events to your OO design.
A: Check if the candidate's listeners can be tied to the method of each object or if they implement global event JS manager functions; also check if the candidate knows about event and OO basics.

Q: How would you design a client application in JavaScript that allows users to simply read and send email messages without a page reload? How could the user be alerted of new messages? Describe the entire interaction flow between your JS code and the server-side application when this happens.
A: The implementation should use Remote Scripting methods (XMLHTTPRequest, document.load, iframes, or other similar approaches). Check to see if they are aware of server-pulling (and other possible approaches).

Q: Say you had to build a client application that composed of various data sets displayed as tables. Next to each set is an "Update" button. Each time the user clicks on the button, that data set is updated. Every request made can take between 1 to 30 seconds. If the user made multiple clicks within a 10 second timeframe, what would happen? (If candidate realizes there would be a problem, ask: How could you solve this problem?)
A: Check to see if the candidate is aware of cuncurrent calls and the need for a dispatch function. If candidate uses the same communication channel, then check if the candidate properly uses lock/unlock or serializing mechanisms, stack calls, or fifo. If the candidate uses concurrent asynchronous channels, then ask the candidate about synchronous approach solutions.

Q: Please describe your solution to the following scenario: Within the browser-frame are present two boxes. One of them moves only on the horizontal axis, while the other only on the vertical axis. As they move through the frame, when they reach either edge they turn around and move in the opposite direction. Additionally if they ever hit each other, again they both reverse direction.
